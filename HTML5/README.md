## About this Accessibility / Flare Project

The project discusses **cognitive load, types of disabilities, assistive technologies,  POUR principles, and WCAG levels of compliance**. We specifically honed in on how chunking, chapters, on-screen markup are significant in the digital world. The project is broken up into sections Cognitive load and accessibility, What is cognitive load?, Reducing cognitive load in a digital space, Assistive technology to reduce cognitive load, How To Add Accessible Features in Clipchamp, How to Add Visual Mark-Up to a Video, Best Practices: Principles for Digital Accessibility, and Tips for Digital Accessibility.

### Tools and concepts used

Tools:

- Clipchamp — web-based video editor
- MadCap Flare — Help Authoring Tool (HAT)
- GitHub - platform for managing project files

Concepts:

- Cognitive load: the mental effort required to process information; reducing it is both a usability improvement and an accessibility requirement
- POUR: the four principles from W3C: Perceivable, Operable, Understandable, Robust
- WCAG levels: A (basic), AA (standard target), AAA (aspirational)
- Chunking / chapters / skimmability: breaking content into manageable segments to support attention and working memory
- Medical vs. social model of disability: medical model frames disability as individual deficit; social model frames it as a failure of environmental design
- Inclusive design vs. universal design vs. retrofitting: proactive vs. reactive approaches to accessibility
- Alt text, captions, transcripts, audio descriptions: text equivalents for non-text content
